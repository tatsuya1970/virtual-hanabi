# バーチャル花火大会 ホームページ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
バーチャル花火大会の公式ホームページを制作し、以下の2つのビジネス目標を達成する。

1. **受注獲得**: 企業・自治体からバーチャル花火大会開催の依頼を獲得する
2. **スポンサー獲得**: 協賛企業・個人から協賛金を獲得する

### 1.2 背景
- 花火大会が中止となった自治体に対し、メタバース/AR連携によるバーチャル開催を提案する事業
- 呉花火大会を起点に5大会・全13回・延べ4,075名の実績がある
- 2026年は呉（7/18）、広島（7/25）、あしだ川（8/15）の開催を予定

### 1.3 運営者情報
- 運営者名: タケムラテックラボ
- 代表者: 武村達也
- 所在地: 広島県広島市中区本通
- メールアドレス: takemura@takemura-techlab.com
- ウェブサイト: https://takemura-techlab.com

## 2. 技術要件

### 2.1 技術構成
| 項目 | 内容 |
|------|------|
| フレームワーク | Next.js (App Router) |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS |
| ホスティング | Vercel |
| ドメイン | virtual-hanabi.com（後日取得・連携） |

### 2.2 対応環境
- PC（Chrome, Safari, Firefox, Edge 最新版）
- スマートフォン（iOS Safari, Android Chrome）
- レスポンシブデザイン必須

### 2.3 パフォーマンス要件
- Lighthouse Performance スコア 90以上を目標
- 画像は WebP / next/image で最適化
- 動画はmp4、遅延読み込み対応

## 3. デザイン要件

### 3.1 デザインコンセプト
- **ベース**: ダーク系（夜空をイメージした背景）
- **アクセントカラー**: オレンジ・金色（花火の色彩）
- **雰囲気**: 華やかさと信頼感の両立。花火の臨場感を伝えつつ、企業・自治体が安心して問い合わせできるプロフェッショナルな印象

### 3.2 ロゴ
- 初期段階ではテキストロゴで実装
- 正式ロゴは後日作成し差し替え

### 3.3 フォント
- 日本語: Noto Sans JP（Google Fonts）
- 見出し等で装飾的なフォントを適宜使用

## 4. ページ構成・セクション設計

シングルページ構成（7セクション）とする。ヘッダーナビゲーションから各セクションへスムーズスクロールで遷移する。

### 4.1 ヘッダー
- ロゴ（テキストロゴ → 後日画像に差し替え）
- ナビゲーション（各セクションへのアンカーリンク）
- スマホ時はハンバーガーメニュー
- スクロール時に背景色を追加（透過 → 半透明ダーク）

### 4.2 セクション1: ヒーロー
- **背景**: 花火の動画または画像（全画面表示）
  - 動画素材は後日配置（初期は静止画でも可）
- **キャッチコピー**: メインコピー + サブコピー（文言は別途決定）
- **実績数字**: カウントアップアニメーションで表示
  - 開催回数: 13回
  - 延べ参加者数: 4,075名
  - 大会数: 5大会
  - 活動年数: 5年間
- **CTAボタン**: 「お問い合わせ」→ お問い合わせセクションへスクロール

### 4.3 セクション2: サービス紹介
以下の3つのサブセクションで構成する。

#### 4.3.1 バーチャル花火大会とは
- メタバース空間で花火大会を開催するサービスの概要説明
- 中止となった花火大会の代替案としての位置づけ
- リアル開催より低コストで実施可能

#### 4.3.2 各大会の独自体験
- 戦艦大和に乗れる（呉）
- 路面電車に乗れる（広島）
- 鞆の街並みのフォトグラメトリ（鞆の浦）
- 廃線となった鞆鉄道に乗れる（あしだ川）
- カード型レイアウトで視覚的に紹介

#### 4.3.3 対応デバイス
- PC（ブラウザ）
- スマートフォン
- VRヘッドセット
- 対応プラットフォーム: cluster

### 4.6 セクション5: 開催プラン（自治体・企業向け）
バーチャル花火大会の開催を検討する自治体・企業向けの情報。

#### 掲載内容
- **開催フロー**: お問い合わせ → ヒアリング → 企画提案 → 制作 → リハーサル → 開催
- **料金**: 10万円〜（規模・内容により変動）
- **訴求ポイント**:
  - 中止の代替案として地域イベントを継続できる
  - リアル開催より低コストで実施可能
  - AR連携による現地ハイブリッド体験
  - 自宅からのオンライン視聴にも対応
  - 地域活性化・観光プロモーションに貢献
- **CTAボタン**: 「開催について相談する」→ お問い合わせセクションへ

## 6. 機能要件

### 6.1 アニメーション・インタラクション
- ヒーロー実績数字のカウントアップアニメーション
- スクロール連動のフェードイン表示（各セクション）
- スムーズスクロール（ナビゲーション）
- ヘッダーの背景変化（スクロール時）

### 6.2 SEO対策
- 適切なmeta title / description の設定
- OGP画像の設定（SNS共有時の表示）
- 構造化データ（JSON-LD）の設定（イベント情報）
- sitemap.xml / robots.txt の生成

### 6.3 アクセシビリティ
- セマンティックなHTML構造
- 画像のalt属性を適切に設定
- キーボードナビゲーション対応
- 適切なコントラスト比の確保

---

## 7. 非機能要件

### 7.1 保守性
- コンポーネント分割による再利用性の確保
- 開催実績・開催予定データは定数ファイルで一元管理（更新容易性）

### 7.2 拡張性
- 新しい大会が追加された際にデータファイルの追記のみで対応可能な構成
- Google Form差し替え・ロゴ差し替えが容易な構成
